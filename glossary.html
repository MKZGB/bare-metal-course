Reference Glossary

Key terms used across the course, defined once for consistency. Terms are grouped by theme; within each group they build on one another.

How to use this page Skim it once before Module 01, then return whenever a term in a module is unfamiliar. Register names link conceptually to the module that uses them.
Architecture & Core

ARM / Cortex-M concepts

RISC (Reduced Instruction Set Computer)

A CPU design philosophy favouring many simple, fixed-length, single-cycle-ish instructions over few complex ones. ARM is a RISC architecture. (Module 01)

Load-store architecture

A RISC trait: arithmetic/logic instructions operate only on registers; memory is touched only by dedicated load (ldr) and store (str) instructions. (Module 01)

Cortex-M3

The ARM processor core inside the STM32F100. Implements the ARMv7-M architecture, runs the Thumb-2 instruction set, is little-endian, and has a 3-stage pipeline. (Modules 01–02)

Thumb / Thumb-2

Compact 16/32-bit instruction encodings that improve code density on flash-constrained MCUs. Cortex-M cores execute Thumb-2 exclusively; the "T" bit in the EPSR must stay 1. (Modules 01–02)

Core registers (R0–R15)

R0–R12 are general-purpose. R13 = SP (stack pointer), R14 = LR (link register, holds the return address), R15 = PC (program counter). (Module 02)

MSP / PSP

The two banked stack pointers selected by R13: the Main Stack Pointer (used at reset and in handlers) and the Process Stack Pointer (often used by thread/task code under an RTOS). This course uses MSP only. (Module 02)

xPSR (APSR / IPSR / EPSR)

The Program Status Register. APSR holds condition flags (N, Z, C, V); IPSR holds the number of the exception currently executing (0 in thread mode); EPSR holds execution state including the Thumb bit. (Module 02)

Thread mode / Handler mode

Thread mode runs normal code; the core enters Handler mode automatically to run an exception/interrupt handler. (Modules 02, 10)

Startup & Memory

Getting from reset to main()

Vector table

An array at the start of Flash (0x08000000) whose entries are the initial stack pointer (entry 0) followed by the addresses of the reset handler and every exception/interrupt handler. The hardware indexes it by exception number. Defined in common/startup.c. (Modules 02, 10)

Reset handler

The first code that runs after reset. It copies initialised data from Flash to RAM, zeroes the .bss section, then calls main(). See Reset_Handler in common/startup.c. (Module 02)

.data / .bss

.data = globals with non-zero initial values (stored in Flash, copied to RAM at startup). .bss = globals initialised to zero (no Flash storage; the startup code zeroes the RAM). (Module 02)

Linker script (linker.ld)

Tells the linker the chip's memory map (Flash at 0x08000000, SRAM at 0x20000000) and which sections go where, including the load vs run address split for .data. (Modules 02–03)

VMA / LMA

Virtual Memory Address (where a section is accessed at run time, e.g. .data in RAM) vs Load Memory Address (where its image is stored, e.g. in Flash). The startup code bridges the two. (Module 02)

Registers & MMIO

Talking to hardware

MMIO (Memory-Mapped I/O)

Peripheral control registers appear at fixed addresses in the CPU's address space; reading/writing those addresses controls the hardware. STM32 peripherals start at 0x40000000; core peripherals (NVIC, SysTick) at 0xE000E000. (Module 03)

volatile

A C qualifier telling the compiler a variable/register may change outside the program's control, so every access must really happen and must not be reordered or cached. Mandatory for MMIO registers and for variables shared with interrupt handlers. (Modules 03, 10)

RCC (Reset and Clock Control)

The peripheral that gates each other peripheral's clock. A peripheral with its clock off ignores all register writes — so you enable its clock in an RCC …ENR register first. Base 0x40021000. (Modules 03–04)

Clock gating

Switching a peripheral's clock on/off to save power. Forgetting to enable a clock is the classic "my driver does nothing" bug. (Module 04)

Read-modify-write

Reading a register, changing some bits with mask/shift operations, and writing it back — used to set a field without disturbing the others. Can race with interrupts, which is why atomic registers like BSRR exist. (Modules 03–04)

GPIO

General-purpose I/O

CRL / CRH

The STM32F1 GPIO configuration registers: CRL configures pins 0–7, CRH pins 8–15, using 4 bits per pin (2 MODE + 2 CNF). The F1 family has no MODER register (that is an F0/F3/F4 feature). (Module 04)

MODE / CNF bits

The 4-bit per-pin field in CRL/CRH: MODE selects input vs output (and output speed); CNF selects the flavour (push-pull / open-drain / alternate-function for outputs; analog / floating / pull-up-down for inputs). (Modules 04–05)

IDR / ODR

Input Data Register (read pin levels) and Output Data Register (drive pin levels). (Modules 04–05)

BSRR (Bit Set/Reset Register)

Writing bit n sets pin n; writing bit n+16 clears it. A single atomic write changes only the pins you name — no read-modify-write, no race. (Module 04)

Alternate Function (AF)

A pin mode (CNF = AF push-pull/open-drain) that hands the pin to a peripheral such as a USART, timer, I²C or SPI instead of plain GPIO. e.g. PA9 as USART1 TX. (Modules 06+)

UART / Serial

Serial communication

USART / UART

(Universal Synchronous/Asynchronous Receiver-Transmitter.) The STM32 serial peripheral. In asynchronous (UART) mode it frames bytes with start/stop bits at an agreed baud rate. USART1 base 0x40013800. (Modules 06–07)

Baud rate / BRR

The bit rate of the serial line. The Baud Rate Register encodes fck / (16 × baud). For 8 MHz / 9600 baud, BRR = 0x0341. (Module 06)

TXE / TC / RXNE

USART status flags: TXE = transmit data register empty (ok to write the next byte); TC = transmission complete (last bit shifted out); RXNE = receive data register not empty (a byte arrived). (Modules 06–07)

Retargeting printf

Redirecting the C library's output to your hardware by implementing the low-level _write() syscall so it sends characters to the UART. With unbuffered stdout this needs no heap. (Module 06)

Semihosting

A debug mechanism where the target asks the host/debugger to perform I/O (e.g. console output) via a special breakpoint. An alternative to UART for seeing output; this course uses UART because it works cleanly under QEMU's serial routing. (Module 06)

Timers & Interrupts

Time and events

SysTick

A simple 24-bit down-counter built into every Cortex-M core (at 0xE000E010), commonly used for a periodic millisecond tick. Registers: CSR (control), RVR (reload), CVR (current). (Module 08)

Prescaler (PSC) / Auto-reload (ARR)

In a general-purpose timer, PSC divides the input clock (by PSC+1) to set the counting rate, and ARR sets the count at which the timer wraps and raises an update event. (Module 09)

Output Compare / PWM

A timer mode that toggles or drives a pin when the counter matches a compare value (CCRx) — used to generate PWM (variable duty-cycle) signals. (Module 09)

Input Capture

A timer mode that latches the counter value into CCRx on an input edge — used to measure pulse widths or signal frequency. (Module 09)

Exception / Interrupt (IRQ)

An event that diverts the CPU to a handler via the vector table. System exceptions (Reset, HardFault, SysTick…) have fixed slots; peripheral interrupts (IRQs) are numbered per RM0041 Table 51. (Module 10)

NVIC (Nested Vectored Interrupt Controller)

The Cortex-M unit that enables/prioritises interrupts. You enable IRQ n by writing bit n%32 of NVIC_ISER[n/32] (base 0xE000E100). It also handles nesting and automatic context save/restore. (Module 10)

EXTI (External Interrupt/Event Controller)

Routes GPIO pin edges (and a few internal sources) to NVIC interrupt lines, with rising/falling edge selection and a pending-bit you clear in the handler. (Module 10)

DMA & Serial Buses

Data movement & device buses

DMA (Direct Memory Access)

A controller that moves data between memory and peripherals without CPU involvement, freeing the core. On the F100, DMA1 has 7 channels; USART1-TX uses channel 4. Base 0x40020000. (Module 11)

I²C (Inter-Integrated Circuit)

A two-wire (SCL clock, SDA data) multi-drop bus where a master addresses 7-bit slave devices with ACK/NACK handshaking and START/STOP conditions. (Module 12)

SPI (Serial Peripheral Interface)

A four-wire full-duplex bus (SCK, MOSI, MISO, chip-select) with no addressing; the master clocks data out and in simultaneously. Clock polarity/phase are set by CPOL/CPHA. (Module 13)

APB1 / APB2 / AHB

The internal peripheral buses. AHB is the fast bus (DMA, the core's GPIO clock tree gating); APB2 hosts higher-speed peripherals (GPIO, USART1, SPI1, TIM1); APB1 hosts the rest (USART2/3, I²C, SPI2, TIM2/3/4). Which bus a peripheral is on determines which RCC …ENR register enables it. (Modules 03+)

Toolchain & QEMU

Building & running

arm-none-eabi-gcc

The bare-metal ("none" OS, EABI) ARM cross-compiler used to build all firmware. Companion tools: objdump (disassemble), objcopy (make a raw binary), readelf (inspect sections), size. (Modules 01–02, all builds)

newlib / nano.specs

The small C library used for embedded targets; --specs=nano.specs selects the size-optimised variant (integer-only printf). --specs=nosys.specs supplies stub OS syscalls you can override. (Module 06)

QEMU -M stm32vldiscovery

The QEMU machine that emulates this board's Cortex-M3 and USARTs. It does not model GPIO, timers, DMA or I²C — see the emulation table in the README. (All modules)

The 3-serial routing rule

Because the machine has three USARTs, QEMU needs a backend for each. Run with -serial stdio -serial null -serial null so USART1 reaches your terminal; a bare -serial stdio errors out. (Module 06)

Arquitetura e Núcleo

Conceitos ARM / Cortex-M

RISC (Reduced Instruction Set Computer)

Uma filosofia de conceção de CPU que favorece muitas instruções simples, de comprimento fixo e quase de ciclo único, em vez de poucas instruções complexas. O ARM é uma arquitetura RISC. (Módulo 01)

Arquitetura load-store

Uma característica RISC: as instruções aritméticas/lógicas operam apenas sobre registos; a memória só é acedida por instruções dedicadas de carregamento (ldr) e armazenamento (str). (Módulo 01)

Cortex-M3

O núcleo de processador ARM dentro do STM32F100. Implementa a arquitetura ARMv7-M, executa o conjunto de instruções Thumb-2, é little-endian e tem um pipeline de 3 andares. (Módulos 01–02)

Thumb / Thumb-2

Codificações de instruções compactas de 16/32 bits que melhoram a densidade de código em MCUs com flash limitada. Os núcleos Cortex-M executam Thumb-2 exclusivamente; o bit "T" no EPSR tem de permanecer a 1. (Módulos 01–02)

Registos do núcleo (R0–R15)

R0–R12 são de uso geral. R13 = SP (apontador de pilha), R14 = LR (registo de ligação, contém o endereço de retorno), R15 = PC (contador de programa). (Módulo 02)

MSP / PSP

Os dois apontadores de pilha alternáveis selecionados por R13: o apontador de pilha principal (Main Stack Pointer, usado no reset e nos handlers) e o apontador de pilha de processo (Process Stack Pointer, frequentemente usado por código de threads/tarefas sob um RTOS). Este curso usa apenas o MSP. (Módulo 02)

xPSR (APSR / IPSR / EPSR)

O registo de estado do programa. O APSR contém as flags de condição (N, Z, C, V); o IPSR contém o número da exceção atualmente em execução (0 em modo thread); o EPSR contém o estado de execução, incluindo o bit Thumb. (Módulo 02)

Modo thread / Modo handler

O modo thread executa código normal; o núcleo entra automaticamente no modo handler para executar um handler de exceção/interrupção. (Módulos 02, 10)

Arranque e Memória

Do reset até main()

Tabela de vetores

Um array no início da Flash (0x08000000) cujas entradas são o apontador de pilha inicial (entrada 0) seguido dos endereços do reset handler e de cada handler de exceção/interrupção. O hardware indexa-a pelo número da exceção. Definida em common/startup.c. (Módulos 02, 10)

Reset handler

O primeiro código que corre após o reset. Copia os dados inicializados da Flash para a RAM, zera a secção .bss e depois chama main(). Ver Reset_Handler em common/startup.c. (Módulo 02)

.data / .bss

.data = variáveis globais com valores iniciais não nulos (armazenadas na Flash, copiadas para a RAM no arranque). .bss = variáveis globais inicializadas a zero (sem armazenamento na Flash; o código de arranque zera a RAM). (Módulo 02)

Script do linker (linker.ld)

Indica ao linker o mapa de memória do chip (Flash em 0x08000000, SRAM em 0x20000000) e que secções vão para onde, incluindo a separação entre endereço de carregamento e de execução para a .data. (Módulos 02–03)

VMA / LMA

Endereço de memória virtual (Virtual Memory Address, onde uma secção é acedida em tempo de execução, p. ex. a .data na RAM) versus endereço de memória de carregamento (Load Memory Address, onde a sua imagem é armazenada, p. ex. na Flash). O código de arranque faz a ponte entre os dois. (Módulo 02)

Registos e MMIO

Comunicar com o hardware

MMIO (Memory-Mapped I/O)

Os registos de controlo dos periféricos aparecem em endereços fixos no espaço de endereçamento do CPU; ler/escrever nesses endereços controla o hardware. Os periféricos do STM32 começam em 0x40000000; os periféricos do núcleo (NVIC, SysTick) em 0xE000E000. (Módulo 03)

volatile

Um qualificador de C que diz ao compilador que uma variável/registo pode mudar fora do controlo do programa, pelo que cada acesso tem de acontecer realmente e não pode ser reordenado nem colocado em cache. Obrigatório para registos MMIO e para variáveis partilhadas com handlers de interrupção. (Módulos 03, 10)

RCC (Reset and Clock Control)

O periférico que controla (gating) o relógio de cada um dos outros periféricos. Um periférico com o relógio desligado ignora todas as escritas nos registos — por isso, primeiro ligas o seu relógio num registo …ENR do RCC. Base 0x40021000. (Módulos 03–04)

Clock gating (gestão de relógio)

Ligar/desligar o relógio de um periférico para poupar energia. Esquecer-se de ligar um relógio é o clássico bug do "o meu driver não faz nada". (Módulo 04)

Read-modify-write (ler-modificar-escrever)

Ler um registo, alterar alguns bits com operações de máscara/deslocamento e voltar a escrevê-lo — usado para definir um campo sem perturbar os restantes. Pode entrar em condição de corrida com interrupções, razão pela qual existem registos atómicos como o BSRR. (Módulos 03–04)

GPIO

Entradas/saídas de uso geral

CRL / CRH

Os registos de configuração do GPIO do STM32F1: o CRL configura os pinos 0–7, o CRH os pinos 8–15, usando 4 bits por pino (2 MODE + 2 CNF). A família F1 não tem registo MODER (isso é uma funcionalidade do F0/F3/F4). (Módulo 04)

Bits MODE / CNF

O campo de 4 bits por pino no CRL/CRH: MODE seleciona entrada vs saída (e a velocidade de saída); CNF seleciona o tipo (push-pull / open-drain / função alternativa para saídas; analógico / flutuante / pull-up-down para entradas). (Módulos 04–05)

IDR / ODR

Input Data Register (lê os níveis dos pinos) e Output Data Register (impõe os níveis dos pinos). (Módulos 04–05)

BSRR (Bit Set/Reset Register)

Escrever o bit n coloca o pino n a 1; escrever o bit n+16 coloca-o a 0. Uma única escrita atómica altera apenas os pinos que indicares — sem read-modify-write, sem condição de corrida. (Módulo 04)

Função alternativa (AF)

Um modo de pino (CNF = AF push-pull/open-drain) que entrega o pino a um periférico, como um USART, timer, I²C ou SPI, em vez de GPIO simples. P. ex. PA9 como TX do USART1. (Módulos 06+)

UART / Série

Comunicação série

USART / UART

(Universal Synchronous/Asynchronous Receiver-Transmitter.) O periférico série do STM32. Em modo assíncrono (UART) enquadra os bytes com bits de início/paragem a um baud rate acordado. Base do USART1 0x40013800. (Módulos 06–07)

Baud rate / BRR

O ritmo de bits da linha série. O Baud Rate Register codifica fck / (16 × baud). Para 8 MHz / 9600 baud, BRR = 0x0341. (Módulo 06)

TXE / TC / RXNE

Flags de estado do USART: TXE = registo de dados de transmissão vazio (pode escrever-se o próximo byte); TC = transmissão completa (último bit enviado); RXNE = registo de dados de receção não vazio (chegou um byte). (Módulos 06–07)

Redirecionar o printf

Redirecionar a saída da biblioteca C para o teu hardware implementando a syscall de baixo nível _write() para que envie caracteres para a UART. Com stdout sem buffer, isto não precisa de heap. (Módulo 06)

Semihosting

Um mecanismo de depuração em que o alvo pede ao host/depurador que execute I/O (p. ex. saída para a consola) através de um breakpoint especial. Uma alternativa à UART para ver a saída; este curso usa UART porque funciona de forma limpa sob o encaminhamento série do QEMU. (Módulo 06)

Timers e Interrupções

Tempo e eventos

SysTick

Um simples contador decrescente de 24 bits incorporado em todos os núcleos Cortex-M (em 0xE000E010), comummente usado para um tick periódico de milissegundo. Registos: CSR (controlo), RVR (recarga), CVR (atual). (Módulo 08)

Prescaler (PSC) / Auto-recarga (ARR)

Num timer de uso geral, o PSC divide o relógio de entrada (por PSC+1) para definir o ritmo de contagem, e o ARR define a contagem à qual o timer dá a volta e gera um evento de atualização. (Módulo 09)

Output Compare / PWM

Um modo de timer que comuta ou aciona um pino quando o contador iguala um valor de comparação (CCRx) — usado para gerar sinais PWM (de duty-cycle variável). (Módulo 09)

Input Capture

Um modo de timer que regista (latches) o valor do contador em CCRx num flanco de entrada — usado para medir larguras de impulso ou a frequência de um sinal. (Módulo 09)

Exceção / Interrupção (IRQ)

Um evento que desvia o CPU para um handler através da tabela de vetores. As exceções de sistema (Reset, HardFault, SysTick…) têm posições fixas; as interrupções de periféricos (IRQs) são numeradas segundo a Tabela 51 do RM0041. (Módulo 10)

NVIC (Nested Vectored Interrupt Controller)

A unidade do Cortex-M que ativa/prioriza interrupções. Ativas a IRQ n escrevendo o bit n%32 de NVIC_ISER[n/32] (base 0xE000E100). Também trata do aninhamento (nesting) e da salvaguarda/restauro automático de contexto. (Módulo 10)

EXTI (External Interrupt/Event Controller)

Encaminha os flancos dos pinos GPIO (e algumas fontes internas) para as linhas de interrupção do NVIC, com seleção de flanco ascendente/descendente e um bit de pendência que limpas no handler. (Módulo 10)

DMA e Barramentos Série

Movimentação de dados e barramentos de dispositivos

DMA (Direct Memory Access)

Um controlador que move dados entre a memória e os periféricos sem envolvimento do CPU, libertando o núcleo. No F100, o DMA1 tem 7 canais; o USART1-TX usa o canal 4. Base 0x40020000. (Módulo 11)

I²C (Inter-Integrated Circuit)

Um barramento multiponto de dois fios (relógio SCL, dados SDA) onde um mestre endereça dispositivos escravos de 7 bits com handshaking ACK/NACK e condições de START/STOP. (Módulo 12)

SPI (Serial Peripheral Interface)

Um barramento full-duplex de quatro fios (SCK, MOSI, MISO, chip-select) sem endereçamento; o mestre desloca dados para fora e para dentro em simultâneo. A polaridade/fase do relógio são definidas por CPOL/CPHA. (Módulo 13)

APB1 / APB2 / AHB

Os barramentos internos de periféricos. O AHB é o barramento rápido (DMA, o gating da árvore de relógio do GPIO do núcleo); o APB2 aloja os periféricos de maior velocidade (GPIO, USART1, SPI1, TIM1); o APB1 aloja os restantes (USART2/3, I²C, SPI2, TIM2/3/4). O barramento em que um periférico se encontra determina qual o registo …ENR do RCC que o ativa. (Módulos 03+)

Toolchain e QEMU

Compilar e executar

arm-none-eabi-gcc

O compilador cruzado ARM bare-metal (SO "none", EABI) usado para compilar todo o firmware. Ferramentas associadas: objdump (desmontar), objcopy (criar um binário em bruto), readelf (inspecionar secções), size. (Módulos 01–02, todas as compilações)

newlib / nano.specs

A pequena biblioteca C usada para alvos embebidos; --specs=nano.specs seleciona a variante otimizada em tamanho (printf só com inteiros). --specs=nosys.specs fornece syscalls de SO em stub que podes substituir. (Módulo 06)

QEMU -M stm32vldiscovery

A máquina QEMU que emula o Cortex-M3 e os USARTs desta placa. Não modela GPIO, timers, DMA nem I²C — ver a tabela de emulação no README. (Todos os módulos)

A regra de encaminhamento dos 3 series

Como a máquina tem três USARTs, o QEMU precisa de um backend para cada um. Executa com -serial stdio -serial null -serial null para que o USART1 chegue ao teu terminal; um simples -serial stdio dá erro. (Módulo 06)
